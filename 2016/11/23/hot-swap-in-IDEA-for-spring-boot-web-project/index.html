IDEA下的Spring-boot web 项目class热加载

发表于 2016-11-23   |   分类于 spring-boot   |  

  普通的java 项目都可以一边修改一边调试，Eclipse等IDE会自动加载新修改的class，但是在Spring-boot中默认却不支持这种开发方式，每次修改代码以后都需要关闭，重启，十分影响效率。下面是两种支持代码热交换的解决方案：

1.springloaded

springloaded采用的是classloader的方式实现类的热加载，非常类似于普通的java项目，可以实现一边修改一边调试。

2. spring-boot-devtools

devtools采用的是自动重启机制，当tools检测到代码更改会自动重启spring的上下文，但由于是热启动，所以速度很快(冷启动8秒，热启动3秒，比springloaded稍慢)。

优点：快速的重启，并且保持重启之前的会话状态，即使手动重启以后，仍然可以保持上一次的会话状态。

缺点：相比springloaded，速度稍慢一点。

http://docs.spring.io/spring-boot/docs/current-SNAPSHOT/reference/htmlsingle/#using-boot-devtools

NOTE:

  1. springloaded和spring-boot-devtools都是监视编译之后的class文件，不是监视java源文件，在eclipse中源文件会自动编译成class文件，但是在IDEA中，需要手动点一下编译按钮，这样java源文件才能编译为class文件。

  2. 如果spring-boot web项目中静态资源文件需要在修改后即时加载，并不需要以上两种方案，spring-boot是默认支持的，对于thymeleaf模板文件需要在application.properties中设置属性spring.thymeleaf.cache=false.
    在IDEA中静态资源文件在修改后也需要点击编译按钮，修改才能即时生效。
